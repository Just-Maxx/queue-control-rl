# Управление очередью методами обучения с подкреплением

Проект посвящен задаче управления очередью с помощью табличных методов обучения с подкреплением.

Рассматривается дискретная стохастическая система с одной очередью и одним сервером. На каждом шаге сервер выбирает один из трех режимов работы:

- `off` — сервер выключен;
- `norm` — обычный режим обслуживания;
- `fast` — ускоренный режим обслуживания.

Состояние системы — текущая длина очереди. Цель управления — минимизировать ожидаемые суммарные затраты, связанные с длиной очереди и стоимостью выбранного режима обслуживания.

## Методы

В проекте реализованы и сравнены:

- Value Iteration;
- Policy Iteration;
- Monte Carlo with Exploring Starts;
- Monte Carlo with ε-greedy policy;
- Off-policy Monte Carlo with Weighted Importance Sampling.

Value Iteration и Policy Iteration используются для получения эталонного решения при известной модели переходов. Методы Монте-Карло обучаются по сгенерированным эпизодам без явного использования таблицы переходов.

## Метрики сравнения

Основная метрика:

```text
mean_value_sse
```

Средняя сумма квадратов отклонений приближенной функции ценности от эталонной функции ценности, найденной методом Value Iteration.

Дополнительная метрика:

```text
mean_policy_match
```

Средняя доля состояний, в которых найденная политика совпадает с эталонной политикой.

## Итоговые результаты

| Метод | mean_value_sse | mean_policy_match | Время, сек. |
|---|---:|---:|---:|
| MC ε-greedy | 2.829 | 0.994 | 21.44 |
| MC Exploring Starts | 3.514 | 0.980 | 22.60 |
| Off-policy MC Weighted IS | 28874.12 | 0.507 | 44.73 |

Лучший результат по основной метрике показал метод Монте-Карло с ε-жадной политикой. Метод с исследовательскими стартами также дал близкий результат. Off-policy метод с weighted importance sampling оказался существенно хуже для данной постановки задачи.


Ноутбук содержит сохраненные результаты и графики. Повторный полный запуск может занять заметное время из-за серий экспериментов и перебора параметров.

## Важные замечания

Ноутбук и папка `src` выполняют разные роли.

`notebooks/queue_control_v4.ipynb` — это оригинальный экспериментальный ноутбук. Он оставлен в исходном виде и не переписывался под импорты из `src`, чтобы не менять сохраненный ход экспериментов.

`src/` — это структурированная версия основного кода, вынесенного из ноутбука для удобного чтения, проверки и переиспользования.

Методы Монте-Карло являются стохастическими, поэтому при повторных запусках возможны небольшие отличия в численных результатах. В коде используются фиксированные random seed, но результаты не обязаны совпадать до последней цифры.
